下面给你一套**从头开始、能落地执行的完整流程**，按你现在这套工作区来。

适用于：
- `HX35HM + SO101 follower + overhead RGB 相机`
- 棋盘格：`7 x 10` 内角点
- 方格边长：`9 mm`
- ArUco 码贴在**活动爪**上
- 推荐先用 `640x480 RGB-only`

---

**总流程**
1. 固定相机
2. 做内参标定
3. 验证内参生效
4. 做外参/手眼标定
5. 写回相机 TF
6. 验证 ArUco 的 `pose_base`
7. 再做红球抓取联调

---

**一、开始前准备**

1. 先把相机支架固定住  
   标定过程中不要再动。

2. 准备棋盘格  
   你的参数是：
   - 内角点：`7 x 10`
   - 方格边长：`9 mm`

3. 准备 ArUco  
   最省事是用**30 mm 边长**的 ArUco，因为当前 detector 默认就是 `0.03 m`。

4. ArUco 固定方式  
   你之前确认过是贴在：
   - `moving_jaw_so101_v1_link`

5. 采外参时注意  
   - 整个采样过程中**夹爪开度不要变**
   - marker 相对活动爪必须刚性固定

**二、内参标定**

**1. 启动 RGB-only 相机**

```bash
cd ~/ros2_ws
source /opt/ros/jazzy/setup.bash
source ~/ros2_ws/install/setup.bash

ros2 launch so101_bringup cameras.launch.py \
  cameras_config:=/home/rog/ros2_ws/src/so101-ros-physical-ai/so101_bringup/config/cameras/so101_cameras_overhead_rgb.yaml
```

用到的配置：
- [so101_cameras_overhead_rgb.yaml](/home/rog/ros2_ws/src/so101-ros-physical-ai/so101_bringup/config/cameras/so101_cameras_overhead_rgb.yaml)

**2. 确认图像话题正常**

```bash
cd ~/ros2_ws
source /opt/ros/jazzy/setup.bash
source ~/ros2_ws/install/setup.bash

ros2 topic hz /static_camera/image_raw
ros2 topic echo /static_camera/camera_info --once
```

**3. 运行内参标定脚本**

```bash
cd ~/ros2_ws
source /opt/ros/jazzy/setup.bash
source ~/ros2_ws/install/setup.bash

python3 ~/ros2_ws/src/so101-ros-physical-ai/tools/camera_intrinsics/calibrate_ros_camera_intrinsics.py \
  --image-topic /static_camera/image_raw \
  --camera-name cam_overhead \
  --cols 7 \
  --rows 10 \
  --square-size-mm 9 \
  --min-samples 18 \
  --output ~/ros2_ws/src/so101-ros-physical-ai/so101_bringup/config/cameras/cam_overhead_calib.yaml
```

脚本：
- [calibrate_ros_camera_intrinsics.py](/home/rog/ros2_ws/src/so101-ros-physical-ai/tools/camera_intrinsics/calibrate_ros_camera_intrinsics.py)

输出文件：
- [cam_overhead_calib.yaml](/home/rog/ros2_ws/src/so101-ros-physical-ai/so101_bringup/config/cameras/cam_overhead_calib.yaml)

**4. 采样方法**
- 画面中心、四角、边缘都要覆盖
- 远近都要有
- 稍微倾斜、旋转，不要全是正对
- 每张姿态明显不同
- 棋盘格不要反光、不要弯

**5. 标定成功后**
你会看到类似：
- `RMS=...`
- `mean_reproj=...`
- `已保存到 ... cam_overhead_calib.yaml`

**6. 重启相机，让新内参生效**

```bash
cd ~/ros2_ws
source /opt/ros/jazzy/setup.bash
source ~/ros2_ws/install/setup.bash

ros2 launch so101_bringup cameras.launch.py \
  cameras_config:=/home/rog/ros2_ws/src/so101-ros-physical-ai/so101_bringup/config/cameras/so101_cameras_overhead_rgb.yaml
```

**7. 验证内参已生效**

```bash
ros2 topic echo /static_camera/camera_info --once
```

确认：
- `width: 640`
- `height: 480`
- `d` 不是全 0
- `k` 是真实值

**六、红球 `x/y` 偏差诊断测试**

这一步的目标不是立刻修，而是先判断：

1. 当前 `x/y` 偏差是不是**固定偏差**
2. 还是随着画面位置变化的**RGB-D 对齐型偏差**

如果你先把这个区分清楚，后面就知道该优先修：
- 外参/补偿表
- 还是 RGB-D 对齐链路

**1. 启动红球检测 + 调试可视化**

```bash
cd ~/ros2_ws
source /opt/ros/jazzy/setup.bash
source ~/ros2_ws/install/setup.bash

ros2 launch so101_bringup follower_hx35hm_moveit.launch.py \
  use_cameras:=true \
  use_rviz:=false \
  use_joint_gui:=false \
  use_aruco_detector:=false \
  use_red_detector:=true \
  use_vision_debug_rviz:=true \
  cameras_config:=/home/rog/ros2_ws/src/so101-ros-physical-ai/so101_bringup/config/cameras/so101_cameras_overhead_rgb.yaml
```

重点观察：
- `/vision/red_block/debug_image`
- `/vision/red_block/pose_base`

**2. 在桌面上准备 5 个测试点**

建议用胶带或纸片做标记：
- 中心 1 个
- 左 1 个
- 右 1 个
- 前 1 个
- 后 1 个

如果方便，再加：
- 左前
- 右前
- 左后
- 右后

这样一共 `5~9` 个点就够了。

**3. 每个点都做同样的记录**

把红球放到某个标记点后：

```bash
ros2 topic echo /vision/red_block/pose_base --once
```

记录两类信息：
- 真实位置
  - 你自己量的桌面点位 `x_real / y_real`
- 检测位置
  - ROS 输出的 `x_detect / y_detect`

建议整理成表格：

```text
点位   x_real   y_real   x_detect   y_detect   dx=x_detect-x_real   dy=y_detect-y_real
P1
P2
P3
...
```

**4. 怎么量真实位置**

尽量统一参考系：
- 以 `base_link` 正下方或机械臂底座中心作为桌面参考原点
- 用卷尺量到红球中心投影点

要求不用特别苛刻，但要前后一致。

**5. 怎么判断偏差类型**

如果你最后看到的是：

**情况 A：几乎所有点都固定偏同一个方向**
- 例如总是：
  - `x` 多 `+1.5 cm`
  - `y` 多 `-0.8 cm`

这更像：
- 外参还有一点固定偏差
- 或者工作区坐标零位有偏差

优先处理：
- 重新微调外参
- 或直接做 `x/y` 常量补偿

**情况 B：越靠画面边缘偏差越大**
- 中间比较准
- 左边偏左、右边偏右
- 前后位置不同，偏差方向也跟着变

这更像：
- RGB 和 depth 没对齐
- 当前“桌面平面投影”近似误差比较大

优先处理：
- 修 RGB-D 对齐
- 修真实深度链路

**情况 C：偏差既有固定偏，也有位置相关变化**

这在你现在这套系统里很常见。通常说明：
- 外参有一点点误差
- 再叠加桌面平面回退模型的近似误差

优先顺序：
1. 先把固定偏差压小
2. 再修 RGB-D 对齐

**6. 一个很实用的经验判断**

如果：
- `debug_image` 里圈球中心一直很准
- 但 `pose_base` 还是在不同位置上有系统偏差

那问题更像是：
- 几何映射 / 外参 / RGB-D 对齐

如果：
- `debug_image` 里球心本身就经常偏

那问题更像是：
- 红球检测本身

**7. 什么时候值得继续修 RGB-D**

如果你做完上面的表，发现：
- 中心点还行
- 越到边缘越偏
- 偏差方向会随着位置变化

那就非常值得继续修 RGB-D 对齐，因为它大概率不只改善 `z`，也会改善 `x/y`。

---

**七、再验证红球**

确认外参没问题后，再开红球 detector。

重点看：
- `/vision/red_block/debug_image`
- `/vision/red_block/pose_base`

如果图像圈得准但 `pose_base` 偏：
- 外参问题或几何映射问题

如果图像本身圈错：
- 红球检测问题

---

**八、你现在最该记住的判断**

如果以后又动了相机：

- 只动了相机安装位置：
  - 只重做**外参**
- 改了分辨率、镜头、焦距：
  - 先重做**内参**
  - 再重做**外参**
